## 🔴 최우선 눈검증 항목 (구현·리뷰에서 도출된 데이터 현실 — 사용자 판단 필요)

### F-1. mid 구간 결제CVR 이상치 오염 (최우선)
- 증상: 유료비중 82~91%(mid) 구간의 실측 CVR 평균이 **9.9%**로, low(5.7%)·high(6.7%)보다 높아 "유료비중↑ → CVR↓" 기대 패턴이 깨짐(비단조).
- 원인: 운영 콘솔 webinars에 **참석 380명 / 구매 362명(CVR 95%)** 인 명백한 입력 오류 행이 mid 구간에 포함되어 평균을 끌어올림(구간 sd도 17.7%p로 비정상적으로 큼).
- 위험: 이 값이 mid 구간 선택 시 "실측 평균 적용됨"(신뢰 라벨)과 함께 제안값으로 채워짐 → **틀린 숫자를 신뢰 라벨로** 보여줄 수 있음.
- 결정 필요: ① 원본 webinars의 해당 행 수정(운영 콘솔에서)  ② 집계 시 이상치 필터 도입(예: CVR 상한 캡)  ③ 그대로 수용. → 사용자 판단.

### F-2. 업종별 결제CVR이 전부 "표본 부족" → ✅ 해결됨(B안 채택, 2026-07-15)
- 결정: **B안(CVR 유료비중 구간 분할 폐지 → 업종별 단일 CVR)** 채택.
- 반영 결과: 결제CVR도 참석률과 동일하게 업종별 집계. **3개 업종(창업·부업 6.7% / 부동산·투자 7.7% / 뷰티·라이프스타일 5.5%)이 실측**, 나머지 5개 소규모 업종만 전체 평균(7.0%) fallback. mix_bucket 관련 코드(Python mix_bucket·JS classifyMixBucket) 전부 제거.
- 남은 사항: F-1(이상치)은 전체 평균 fallback에만 영향(3개 실측 업종은 무관 — 오류행이 industry=null이라 애초 미포함).
