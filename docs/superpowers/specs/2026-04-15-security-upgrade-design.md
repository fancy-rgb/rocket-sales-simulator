# 시뮬레이터 보안 업그레이드 설계 (v1.2.0)

**날짜**: 2026-04-15  
**목표**: 내부 전용 서비스에 구글 로그인 기반 접근 제어 추가

---

## 배경

현재 서비스는 GitHub Pages에 공개 URL로 배포되어 있으며, 링크를 아는 누구나 시나리오를 조회·저장·삭제할 수 있다. Firestore Security Rules도 테스트 모드(누구나 읽기/쓰기 가능) 상태일 가능성이 높다. 내부 파트너 런칭 수치를 다루는 서비스이므로 접근 제어가 필요하다.

---

## 확정 설계

### 1단계 — Firebase Authentication (구글 로그인)

- 앱 진입 시 구글 로그인 화면 표시 (미로그인 상태)
- 로그인 후 도메인 검증:
  - 허용: `@liveklass.com`, `@futureschole.com`
  - 이외 도메인 → 즉시 로그아웃 + 안내 메시지
- `blocked_users` Firestore 컬렉션 조회:
  - 해당 이메일이 있으면 → 즉시 로그아웃 + 안내 메시지
  - 없으면 → 정상 접근
- 헤더에 로그인한 계정 표시 이름 + 로그아웃 버튼 추가

### 2단계 — 작성자 로그인 연동

- 저장 모달의 작성자 드롭다운 → 제거
- 작성자 자동 지정: `user.displayName` (구글 계정 표시 이름)
- 시나리오 카드 작성자 배지는 기존 그대로 유지
- 기존 저장된 시나리오 데이터 변경 없음

### 3단계 — Firestore Security Rules 업데이트

- 현재: `allow read, write: if true` (테스트 모드)
- 변경: `allow read, write: if request.auth != null` (인증된 사용자만)
- Firebase 콘솔에서 직접 수정

---

## 퇴사자 접근 차단

앱 내 관리 UI 없음. Firebase 콘솔에서 직접 관리.  
절차는 `CLAUDE.md > 보안 운영 가이드 > 퇴사자 접근 차단 절차` 참고.

---

## 변경되지 않는 것

- GitHub Pages 배포 방식
- 기존 시뮬레이션 / 실적 입력 기능 전체
- Firestore `scenarios` 컬렉션 구조
- 시나리오 카드 UI

---

## 구현 순서 원칙

1단계 → 검증 → 2단계 → 검증 → 3단계 → 검증  
각 단계 완료 후 실제 동작 확인 후 다음 단계 진행.
