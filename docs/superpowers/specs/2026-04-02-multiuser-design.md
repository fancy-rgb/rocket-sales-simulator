# 멀티유저 시뮬레이터 설계

**날짜**: 2026-04-02
**목표**: 여러 팀원이 동시에 접속하고, 저장한 시나리오를 서로 볼 수 있는 공유 서비스로 전환

---

## 결정 사항

### 공유 방식
- **완전 공유 풀**: 누가 저장하든 모든 시나리오가 하나의 목록에 표시됨
- 각 시나리오 카드에 작성자 이름 배지 표시

### 사용자 식별
- 로그인/회원가입 없음
- 시나리오 저장 시 팀원 목록 드롭다운에서 이름 선택
- 본인 확인 없이 신뢰 기반 운영 (내부 팀 사용, Notion과 동일 방식)

### 저장소
- **Firebase Firestore** (Google 무료 플랜)
- 서버 없음, 별도 배포 없음 — 기존 GitHub Pages 그대로 유지

---

## 데이터 구조

### `scenarios` 컬렉션
시나리오 저장 시 생성되는 문서:

| 필드 | 설명 | 예시 |
|------|------|------|
| `name` | 시나리오 이름 | "박찬희 런칭 낙관 시나리오" |
| `launchName` | 런칭명 | "박찬희 런칭" |
| `author` | 작성자 | "서유진" |
| `type` | 종류 | `"simulation"` 또는 `"actual"` |
| `date` | 저장일 | "2026. 4. 2." |
| `...` | 기존 시뮬레이션 수치 필드들 | (기존 데이터 구조 그대로) |

### `team_members` 컬렉션
팀원 목록 (초기엔 코드에 하드코딩):

| 필드 | 예시 |
|------|------|
| `name` | "서유진", "김민준", ... |

## UI 변경 사항

1. **시나리오 저장 모달**
   - 기존: 이름 텍스트 입력
   - 변경: 팀원 드롭다운 선택

2. **시나리오 카드**
   - 추가: 작성자 이름 배지

3. **데이터 저장/로드**
   - 기존: `localStorage`
   - 변경: Firebase Firestore
   - 동기화 방식: 탭 이동 또는 새로고침 시 최신 반영 (실시간 아님)

---

## Firebase 설정 순서 (약 15분)

1. [console.firebase.google.com](https://console.firebase.google.com) → 구글 계정으로 접속
2. 새 프로젝트 생성
3. Firestore Database 활성화 (테스트 모드로 시작)
4. 앱 등록 → 연결 설정값(config) 복사
5. `index.html`에 Firebase SDK + config 추가
6. 팀원 이름 초기 데이터 Firestore에 입력
7. GitHub에 push → 기존 링크로 자동 배포

- 실시간 동기화 (다른 사람이 저장 시 즉시 반영) — 새로고침으로 충분
