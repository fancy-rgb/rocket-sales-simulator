# 시뮬레이터 UX 시각 개선 우선 — 운영 콘솔 통합은 보류

날짜: 2026-06-02
상태: 확정

## 결정한 내용
매출 시뮬레이터를 운영 콘솔(rocket-launch-dashboard)에 통합하지 않고 **보류**한다.
대신 **UX 시각 개선을 현재 단일 HTML 상태 그대로 먼저** 진행한다.
파트너 노출은 현행대로 **화면 캡처 방식**을 유지한다 (파트너 직접 접근 없음 → 내부 전용 확정).

## 이유
- UX 고통(퍼널·시나리오 비교 가독성)은 파트너 캡처에 즉시 영향 → 빨리 해소하는 게 실익.
- 운영 콘솔에는 진행 중인 P0(테스트 환경 분리)가 있어 통합은 그 이후가 적절.
- 시뮬레이터를 매니저들이 활발히 사용 중 → 입력 방식이 급변하면 사용성 리스크.

## 검토한 대안
- **즉시 전면 통합(React/shadcn 재작성)**: 비용·시기 부적합. UX 개선이 재작성에 묻힐 위험(challenger #1).
- **iframe 점진 임베드**: 네비·인증 일원화는 빠르나, 지금은 UX가 우선이라 향후 옵션으로 보류.

## 영향 범위
- 이번 작업 = 단일 index.html의 **표현/시각 개선만** (입력·계산·데이터·기능 불변).
- 운영 콘솔 통합(`/tools/sales-simulator`), 보수/현실/이상 3안 구조는 **향후 별도 의제**.
  통합 방향 자체는 타당(인증 게이트 공유·"런칭 도구" 섹션 존재·백로그 L8).
